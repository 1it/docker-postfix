## v0.1.3

### Security

- **Upgraded base image to Debian 13 (Trixie)** — from Bookworm, pulling in newer patched packages (postfix 3.10, OpenDKIM 2.11) and cutting the CVE surface.
- **Replaced certbot with acme.sh** — Let's Encrypt issuance now uses [acme.sh](https://github.com/acmesh-official/acme.sh), a single POSIX shell script with no Python dependency stack. This removes `urllib3`, `cryptography`, `requests`, `pyopenssl`, and the rest of certbot's transitive Python packages from the image, eliminating the bulk of previously reported CVEs. None of those packages were used by Postfix or OpenDKIM.
- **Base package upgrade** — `apt-get upgrade` now runs in both build stages so images ship the latest patched Debian point releases (perl, glibc, gnutls28, ...).

### New Features

- **Authenticated submission** — `SMTP_AUTH_USERS` provisions SASL accounts at startup (`user1:pass1,user2:pass2`), enabling external clients to authenticate on the submission port (587). Previously SASL was enabled but no accounts could be created, so authenticated submission always failed and only `MY_NETWORKS` clients could send.
- **Let's Encrypt auto-renewal** — A background loop runs `acme.sh --cron` every 12h and reloads Postfix on renewal. acme.sh state is stored on the `letsencrypt` volume so the account and certificates persist across restarts.
- **Multi-arch publishing to two registries** — CI now builds `linux/amd64` and `linux/arm64` and pushes to Docker Hub (`01it/postfix`, primary) and `ghcr.io` (backup).

### Bug Fixes

- **Removed unserved port 465** — 465/tcp was exposed and documented but no `smtps` service was ever defined in `master.cf`, so the port never listened. Removed from the image and docs.
- **Fixed certificate OU corruption** — `certgen.sh` substituted `SSL_ORGANIZATION` before `SSL_ORGANIZATIONAL_UNIT`, clobbering the OU field. Substitutions are now ordered longest-token-first.
- **Fixed DOMAIN/MAILNAME propagation to cert generation** — `certgen.sh` never received a user-set `DOMAIN` (values were not exported), so certificates always used the generator's built-in defaults. The entrypoint now exports both.
- **Aligned certificate identity** — the self-signed cert CN is now based on `MAILNAME` (Postfix `myhostname`), and `MAILNAME` is added to the certificate SANs.
- **Idempotent restarts** — DKIM extra-domain and trusted-network entries are now guarded, so container restarts no longer accumulate duplicate lines in `signing.table`, `key.table`, and `trusted.hosts`.
- **Trimmed OpenDKIM config** — dropped redundant `Domain`/`KeyFile`/`Selector` directives; signing is driven entirely by the SigningTable/KeyTable.

### New Environment Variables

| Variable | Description |
|----------|-------------|
| `SMTP_AUTH_USERS` | Submission (587) login accounts, `user1:pass1,user2:pass2` |

### Upgrade Notes

- **Port 465** is no longer exposed. If you mapped it, remove that mapping; use 587 for authenticated submission.
- **Docker Hub + ghcr.io** — CI publishing requires `DOCKERHUB_USERNAME` and `DOCKERHUB_TOKEN` repository secrets.
- **Let's Encrypt** now issues via acme.sh over HTTP-01 (standalone, port 80). Keep the `letsencrypt` volume mounted to persist the acme.sh account and certificates.
