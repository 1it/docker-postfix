## v0.1.1

### New Features

- **Direct delivery mode** — Postfix now delivers mail directly to recipient MX servers over port 25 (default when `RELAY_HOST` is not set)
- **Embedded OpenDKIM** — Automatic DKIM key generation and signing for outbound mail; public key printed to stdout on first startup
- **Multi-domain DKIM** — Sign mail for additional domains via `DKIM_EXTRA_DOMAINS`
- **Let's Encrypt support** — Optional automatic TLS certificate issuance via `LETSENCRYPT_EMAIL`, with fallback to self-signed certificates
- **Extra certificate SANs** — Add additional domains to Let's Encrypt certificates via `LETSENCRYPT_EXTRA_DOMAINS`

### Improvements

- Proper sender restrictions (`reject_non_fqdn_sender`, `reject_unknown_sender_domain`)
- Proper recipient restrictions (`reject_unauth_pipelining`, `reject_non_fqdn_recipient`)
- Mode-aware outbound TLS: opportunistic (`may`) for direct delivery, mandatory (`encrypt`) for relay mode
- DNS lookups enabled by default for MX resolution
- Removed obsolete DH parameter generation (unnecessary for TLS 1.2 ECDHE / TLS 1.3)
- Added `recipient_delimiter = +` for address extension support

### New Environment Variables

| Variable | Description |
|----------|-------------|
| `DOMAIN` | Domain for DKIM signing and certificate generation |
| `DKIM_SELECTOR` | DKIM selector (default: `mail`) |
| `DKIM_KEY_DIR` | DKIM key storage directory |
| `DKIM_EXTRA_DOMAINS` | Comma-separated extra domains to DKIM-sign |
| `LETSENCRYPT_EMAIL` | Enables Let's Encrypt certificate issuance |
| `LETSENCRYPT_EXTRA_DOMAINS` | Extra domains for the Let's Encrypt certificate |

### New Volumes

| Volume | Description |
|--------|-------------|
| `dkim_keys` (`/etc/ssl/dkim`) | Persistent DKIM key pair |
| `letsencrypt` (`/etc/letsencrypt`) | Let's Encrypt certificates (optional) |

### Breaking Changes

None. Existing `RELAY_HOST` / `SMTP_USERNAME` / `SMTP_PASSWORD` configuration continues to work as before.
